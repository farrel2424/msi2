Environment Setup

üìã Quick Start

The system requires two API keys to function. You can provide them either through the web interface or by setting environment variables (recommended for security).

Set Environment Variables

For better security, set these variables in your terminal before starting the web UI:

export OPENAI_API_KEY="sk-your-openai-api-key-here" export API_BEARER_TOKEN="your-bearer-token-here"

Or Use a .env File

Create a .env file in the project directory:

OPENAI_API_KEY=sk-your-openai-api-key-here API_BEARER_TOKEN=your-bearer-token-here

Or Provide via Web Interface

Enter the keys directly in the upload form (less secure, but convenient for testing).

Configuration Parameters

OpenAI API Key REQUIRED

Environment Variable: OPENAI_API_KEY

Your OpenAI API key for accessing GPT-4. Get one from https://platform.openai.com/api-keys

Example: sk-proj-abc123...

API Bearer Token REQUIRED

Environment Variable: API_BEARER_TOKEN

Authentication token for your internal API endpoint where extracted data will be submitted.

Example: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

API Endpoint OPTIONAL

Default: https://internal-api.example.com/upload

The URL where extracted JSON data will be POST-ed. Can be configured per-upload in the web interface.

OpenAI Model OPTIONAL

Default: gpt-4

The GPT model to use for extraction. Options: gpt-4, gpt-4-turbo-preview, gpt-4o

Max Retries OPTIONAL

Default: 3

Number of retry attempts for validation failures and API submission errors. Includes automatic self-correction with error feedback.
